Engenharia de Prompts

Base mínima — todas as outras habilidades pressupõem esta
▸

Caminho de aprendizado

  • Ler os guias gratuitos de prompting da Anthropic e da OpenAI.
  • Dedicar uma semana reescrevendo seus piores prompts.

Técnicas centrais

  • Atribuição de função (dizer ao modelo quem ele é e para quem escreve).
  • Prompts com poucos exemplos (few-shot).
  • Raciocínio passo a passo antes de responder.
  • Estrutura de saída definida.
Projeto para iniciantes: assistente de personalização de currículos — usa um prompt bem elaborado para reescrever seu currículo conforme a descrição de vaga inserida.
Critério de conclusão: o assistente reescreve o currículo de forma consistente para diferentes vagas, usando as 4 técnicas acima.

Automação com IA

Maior impacto por esforço — sem programar
▸

Caminho de aprendizado

  • Escolher o n8n (indicado no roteiro por ser open-source e auto-hospedável).
  • Seguir um tutorial gratuito no YouTube.
  • Automatizar uma tarefa real do seu dia a dia — problemas reais ensinam mais rápido que tutoriais.

Áreas com retorno mais claro (citadas no texto)

  • Triagem de e-mails e redação de respostas
  • Qualificação de leads e atualizações de CRM
  • Extração de dados de faturas e recibos
  • Fluxos de reaproveitamento de conteúdo
Projeto para iniciantes: automação de e-mail que lê mensagens recebidas e envia um resumo diário.
Critério de conclusão: a automação roda sem sua intervenção manual.
3

Usando APIs de IA

Aqui entra o Python básico
▸

Caminho de aprendizado

  • Aprender Python básico em um curso para iniciantes no YouTube.
  • Seguir a documentação de início rápido da API Claude ou OpenAI.
  • Opcional: usar um editor de código de IA (ex.: Cursor) para facilitar.

Conceitos essenciais

  • Chaves de API
  • Prompts do sistema
  • Parâmetros (temperatura, tokens)
  • Saídas estruturadas (JSON limpo)
Projeto para iniciantes: resumidor de linha de comando que recebe a URL de um artigo e retorna os pontos principais.
Critério de conclusão: script funcional rodando via terminal (o documento estima menos de uma hora para isso).
4

Fluxos de Trabalho de IA

De um "truque" para um "sistema"
▸

Conceito central

  • Encadear várias etapas em um pipeline, cada uma com função específica — em vez de um único prompt sobrecarregado.
  • Exemplo do texto: extrair afirmações → verificar cada uma → redigir rascunho → reescrever com linguagem da marca.

Dois hábitos cruciais

  • Roteamento: entradas fáceis vão para modelo barato, casos difíceis para modelo caro.
  • Pontos de verificação humana: aprovação humana antes de saídas importantes.

Ferramentas

  • Python com frameworks como LangChain
  • Ou visualmente no n8n (mesma ferramenta da etapa 2)
Projeto para iniciantes: pipeline de reaproveitamento que transforma uma postagem de blog em post de LinkedIn, sequência de tweets e resumo de newsletter — cada saída como etapa própria.
Critério de conclusão: etapas testáveis e depuráveis separadamente (não um prompt único), com o pipeline de reaproveitamento funcional.
Etapas 5–8 · você se torna um construtor
5

Modelos de Código Aberto

Ter o modelo em mãos, não alugado
▸

O que são

  • Modelos de peso aberto que podem ser baixados e executados por você mesmo: Llama (Meta), DeepSeek, Qwen (Alibaba), Gemma (Google), Mistral.
  • Conceitos a entender: inferência, contagem de parâmetros e quantização (compressão do modelo para caber em hardware comum com perda mínima de qualidade).

Por que empresas se importam

  • Privacidade — equipes de saúde, finanças e jurídico muitas vezes não podem enviar dados a uma API externa.
  • Custo — em alto volume, hospedagem própria pode reduzir significativamente o preço por token.

Caminho de aprendizado

  • Instalar o Ollama e rodar um comando para ter um modelo local.
  • Alternativa com interface amigável: LM Studio.
  • Navegar pelo Hugging Face, o hub onde quase todos os modelos abertos estão disponíveis.
Projeto para iniciantes: assistente de notas totalmente privado que responde perguntas sobre seus arquivos sem que nada saia do seu computador.
Critério de conclusão: um modelo 8B quantizado rodando localmente, respondendo com base nos seus próprios arquivos, sem chamadas externas.
6

RAG (Retrieval Augmented Generation)

A habilidade individual mais requisitada no mercado, segundo o texto
▸

Conceito central

  • Permite que o modelo responda usando conhecimento com o qual nunca foi treinado: documentos da empresa, PDFs, dados recentes.
  • Em vez de esperar que o modelo tenha memorizado algo, você recupera o texto relevante e o fornece como contexto no momento da pergunta.

Dois conceitos que sustentam

  • Embeddings: convertem texto em números que capturam significado (ex.: "como recebo meu dinheiro de volta?" e "política de reembolso" ficam próximos, mesmo sem palavras em comum).
  • Banco de dados de vetores (Pinecone, Chroma, Qdrant ou pgvector no Postgres): armazena esses números e encontra correspondências mais próximas em milissegundos.

Pipeline em 4 etapas

  • Dividir documentos em partes (chunking)
  • Converter cada parte em um embedding
  • Armazenar os embeddings em um banco de dados vetorial
  • Recuperar as partes mais relevantes para cada pergunta e passá-las ao modelo

O que separa júnior de sênior

  • Estratégia de segmentação (chunking), busca híbrida (palavra-chave + semântica) e reclassificação (reranking) dos resultados.

Caminho de aprendizado

  • Construir um pipeline em Python puro antes de usar um framework, para entender cada etapa.
  • Depois, explorar o LlamaIndex — o framework mais completo em recuperação de documentos.
Projeto para iniciantes: assistente de PDF — faça upload de um documento, faça perguntas e obtenha respostas citando a passagem original.
Critério de conclusão: as 4 etapas do pipeline funcionando de ponta a ponta, com respostas ancoradas na passagem original recuperada (não inventadas).
7

Ajuste Fino (Fine-Tuning)

Mudar o comportamento do modelo, não só o que ele sabe
▸

Conceito central

  • Treina um modelo existente com seus próprios exemplos, alterando seu comportamento permanentemente: estilo, formato, terminologia de domínio.
  • Se o RAG dá ao modelo uma biblioteca de referência, o ajuste fino o envia para um campo de treinamento.

Quando de fato é necessário (3 situações citadas)

  • Você precisa de estilo ou formato consistente em grande escala.
  • Você quer um modelo pequeno e barato competindo com um modelo grande em tarefa específica.
  • Suas instruções (prompts) se tornaram monstros incontroláveis.

Técnica e ferramentas

  • LoRA (e sua variante econômica em memória, QLoRA): treina um pequeno adaptador sobre um modelo congelado, em vez de retreinar tudo.
  • Poucas centenas a alguns milhares de exemplos de qualidade + GPU alugada + algumas horas. A qualidade do conjunto de dados é o fator decisivo.
  • Ferramentas: Hugging Face TRL, Axolotl, Unsloth, GPUs alugadas via Google Colab ou RunPod.
Projeto para iniciantes: ajustar um pequeno modelo Llama ou Qwen com 500 amostras da sua própria escrita, até que ele escreva com a sua voz.
Critério de conclusão: o modelo ajustado reproduz de forma consistente seu estilo/voz nas saídas, usando LoRA/QLoRA sobre um modelo aberto.
8

Agentes de IA

De responder para agir
▸

Conceito central

  • Um agente é um modelo em loop com ferramentas: você define um objetivo, ele decide ações, observa resultados e continua até concluir.
  • Um fluxo de trabalho segue sua receita; um agente recebe a cozinha e é instruído a preparar o jantar.
  • Ferramentas são funções que o modelo pode chamar: pesquisar na web, consultar banco de dados, enviar e-mail, executar código.

MCP (Model Context Protocol)

  • Padrão aberto que permite que qualquer agente se conecte a qualquer ferramenta sem código de integração personalizado.
  • Downloads do SDK cresceram de ~2 milhões para 97 milhões por mês em dezesseis meses; milhares de servidores MCP públicos hoje cobrem bancos de dados, CRMs e ferramentas de produtividade.

Alerta do texto

  • Agentes falham de maneiras criativas; a maioria dos "agentes" em produção são fluxos de trabalho limitados com poucas etapas de intervenção do agente. Saber quando não usar um agente faz parte da habilidade.

Caminho de aprendizado

  • Construir primeiro um loop de agente básico em Python (modelo, ferramentas, loop — cerca de 100 linhas), para que nada seja mágico.
  • Depois escolher um framework: LangGraph (controle), CrewAI (velocidade) ou SDK de Agentes Claude (integração profunda com MCP).
Projeto para iniciantes: agente de pesquisa que recebe um tópico, pesquisa na web, lê fontes e escreve um relatório citado.
Critério de conclusão: o loop básico (modelo + ferramentas + observação + continuação) funciona sem lógica "mágica" escondida em um framework.

Sistemas Multiagentes

Equipes de especialistas em vez de um generalista sobrecarregado
▸

Conceito central

  • Divide o trabalho entre agentes especializados: um planejador decompõe a tarefa, um pesquisador coleta informações, um redator elabora o rascunho, um revisor faz a crítica.

Padrões de coordenação

  • Supervisor: um agente delega o trabalho e reúne os resultados.
  • Pipeline: o trabalho flui em linha, cada agente transformando-o.
  • Debate: os agentes criticam o trabalho uns dos outros para detectar erros.

Protocolo A2A

  • Junto ao MCP (para ferramentas), o protocolo A2A (agente para agente) está emergindo como padrão para agentes que delegam tarefas entre equipes e até empresas.

Alerta do texto

  • Cada agente adicional aumenta custo, latência e falhas. Regra prática da indústria: use o menor número possível de agentes que funcionem.

Ferramentas

  • CrewAI para protótipos rápidos baseados em funções.
  • LangGraph para controle preciso.
Projeto para iniciantes: estúdio de conteúdo com três agentes (pesquisador, redator, editor) produzindo um artigo finalizado a partir de um briefing de uma linha.
Critério de conclusão: os três agentes têm papéis e contextos bem definidos, coordenados por um dos padrões (supervisor, pipeline ou debate).

Avaliação LLM

Habilidade pouco glamorosa, entre as mais raras e bem recompensadas
▸

Conceito central

  • Mede se um sistema de IA é realmente bom e se a última alteração o melhorou ou piorou — como um teste unitário para um mundo onde a mesma entrada pode gerar saídas diferentes.

Fluxo de trabalho (3 passos)

  • Construir um conjunto de dados com casos de teste realistas.
  • Definir o que seria uma boa resposta para cada caso.
  • Executar cada alteração no conjunto de dados e comparar as pontuações.

Como pontuar

  • Verificações de código (JSON válido? citação real?), revisão humana e avaliação por um LLM segundo uma rubrica — sempre verificando os avaliadores automáticos contra classificações humanas.

Caminho de aprendizado

  • Escrever 30 casos de teste para qualquer projeto desta trilha.
  • Avaliar as saídas manualmente primeiro.
  • Depois automatizar com um avaliador LLM e comparar os resultados.
  • Explorar LangSmith, Langfuse (open source) ou DeepEval.
Projeto para iniciantes: conjunto completo de avaliações para o assistente de PDF (etapa 6), medindo precisão e verificando se as respostas permanecem fiéis à fonte recuperada.
Critério de conclusão: você consegue provar, com números, se uma alteração no sistema melhorou ou piorou o resultado — não apenas "parece funcionar".
11

Implantação do LLM

De "funciona no meu laptop" para "funciona para mil usuários"
▸

Conceito central

  • Transforma o projeto em um serviço ativo: um endpoint de API, um aplicativo web, algo com uma URL.

Dois caminhos

  • Simples: encapsular uma API de modelo hospedada em seu próprio backend (FastAPI é o padrão em Python), adicionar autenticação e limitação de taxa, implantar em uma plataforma de nuvem.
  • Avançado: servir um modelo de código aberto por conta própria usando vLLM, o mecanismo de inferência que agrupa solicitações para extrair o máximo de desempenho de uma GPU.

Detalhes que separam amador de profissional

  • Respostas em fluxo contínuo (streaming), soluções alternativas para interrupções do provedor, limites de custo por usuário.

Ferramentas

  • FastAPI, Docker, vLLM, plataformas em nuvem como Railway ou AWS, Streamlit para interfaces rápidas.
Projeto para iniciantes: implantar o assistente de PDF avaliado (etapas 6 e 10) como um aplicativo web público, com login e limite de uso.
Critério de conclusão: o aplicativo está acessível publicamente por uma URL, com autenticação e algum limite de uso — não apenas rodando localmente.
12

LLMOps

Operar de forma lucrativa por anos, não só abrir o restaurante
▸

Conceito central

  • Engloba tudo o que é necessário para operar sistemas de IA em produção ao longo do tempo.

Cinco frentes do dia a dia

  • Rastreamento: cada solicitação é registrada para reproduzir exatamente o que o modelo viu e fez.
  • Painéis: custo, latência e taxas de erro monitorados continuamente.
  • Avaliação online: amostra do tráfego ao vivo pontuada automaticamente (conecta-se diretamente à etapa 10).
  • Versionamento de prompts: prompts tratados como código, já que uma pequena alteração pode quebrar o comportamento silenciosamente.
  • Proteções (guardrails): filtros de entrada e saída e, com agentes, autorização para chamadas de ferramentas.

Ferramentas

  • Langfuse ou LangSmith para rastreamento e monitoramento, Helicone para visibilidade de custos, além de Docker e CI/CD básico.
Projeto final: dar ao aplicativo implantado (etapa 11) todos os recursos de produção: rastreamento em cada solicitação, painel de custos, avaliações automatizadas em tráfego real e um alerta quando a qualidade cair.
Critério de conclusão: as cinco frentes (rastreamento, painéis, avaliação online, versionamento de prompts, proteções) estão implementadas e conectadas ao mesmo aplicativo.
